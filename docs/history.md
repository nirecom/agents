### FEATURE: enforce-worktree blocks memory directory writes from main worktree (2026-05-26, 61699cdf7a59ef2b6f37afa940211c0898f9e913, #573)
Background: `~/.claude/projects/<repo>/memory/` is Claude Code's auto-memory directory (per the global `# auto memory` instructions). It lives outside any git repo and is conceptually a per-user state store. It should be freely readable and writable regardless of which worktree or branch is currently checked out. Currently `hooks/enforce-worktree.js` blocks Bash write operations against this directory whenever the session CWD is the main worktree, because the guard is CWD/branch-based and does not exempt paths outside the agents repo. This is structurally the same class of bug as #419 / #497 (`~/.workflow-plans/` blocked despite being outside the repo). Filed separately because the memory path was never explicitly covered and the asymmetry below adds a new failure mode. ### Example (encountered 2026-05-26) After /worktree-end finished and CWD was switched to the main worktree (`C:/git/agents` on branch 'main'), Claude wrote a feedback memory file via the Write tool (which is path-classified differently and succeeded). The user then said 'don't write to memory'. Claude tried to delete it with: ``` rm "C:/Users/nire/.claude/projects/c--git-agents/memory/feedback_final_report_surface_restart.md" ``` enforce-worktree blocked the rm with: ``` ENFORCE_WORKTREE: write blocked. Reason: main worktree (branch 'main'). ``` Workarounds available to Claude were all heavy: - ask user to delete manually via PowerShell - ask user to approve `<<WORKFLOW_ENFORCE_WORKFLOW_OFF>>` just to delete one file - /sweep-worktrees does not cover memory files (out of scope) **Asymmetry**: Write tool can create memory files from the main worktree, but Bash rm cannot delete them. So Claude can dig itself into a hole it cannot exit without escalation.
Changes: - Exempt the memory directory tree (`~/.claude/projects/*/memory/**` or equivalent platform path) from enforce-worktree's write-block, regardless of CWD/branch. - Apply the exemption uniformly across Bash write operations (rm, mv, cp, redirect, etc.) so behavior matches the Write tool's existing pass-through. - Verify enforce-worktree path resolution handles Windows backslash / mixed-separator paths correctly for this exemption. - Consider whether a general 'non-repo path exemption' (per §1/§2 of core-principles) would cover both `~/.claude/projects/*/memory/` and the already-fixed `~/.workflow-plans/` from one place, rather than per-path patches. - Add a test covering: from main worktree CWD, both Write-tool create and Bash rm against a memory file succeed.
