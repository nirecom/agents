### FEATURE: PR #816 — fix/fix-802 (2026-06-08, e5c2e55297995e1123391303e367371648fd8458, #816)

Background: fix(#802): block interpreter-wrapper bypass of isAllowedWorktreeCommand

Changes: BUGFIX (#802): `bash -c 'git worktree remove X && rm -rf /'` from the main worktree was incorrectly allowed by `isAllowedWorktreeCommand`. Root cause: `hasShellChaining(cmd)` invokes `stripQuotedArgs(cmd)` first, collapsing the single-quoted body of `bash -c '...'` to `''`. The chaining detector then saw `bash -c ''` (no operators) and returned false, after which the predicate matched `git ... worktree remove` substrings inside the SQ literal on the raw cmd and returned true. Fix: added two guards to `isAllowedWorktreeCommand` in `hooks/enforce-worktree/main-worktree-allows.js`. Guard 1 is an interpreter-prefix reject placed as the first statement in the function — matches `^\s*(?:[A-Z_][A-Z0-9_]*=\S*\s+)*\b(bash|sh|zsh|dash|pwsh|powershell|cmd|node|python|perl|ruby)\b` and returns false, blocking direct interpreter wrappers plus uppercase env-prefix variants like `WORKTREE_END_SKILL=1 bash -c '...'`. Guard 2 is a belt-and-braces complete-shape gate placed after the existing git/worktree match — runs `stripQuotedArgs(cmd)` and rejects any residual `|`, `;`, `&`, `$(`, or backtick to document intent and guard against future `hasShellChaining` refactors. Test coverage: 4 new E2E cases (E-WT-REMOVE-4..7 in `tests/fix-595-bypass-e2e.sh`) and 4 new unit cases (S45..S48 in `tests/fix-enforce-worktree-main-cleanup.sh`); direct `git worktree remove "$path"` with DQ-quoted paths remains allowed (regression pin). Sibling allow predicates (`isAllowedFastForwardMerge`, `isAllowedMainWorktreeCleanup`, `isAllowedPushAllExcluded`) share the same vulnerability shape and are deferred to a follow-up issue per Accepted Tradeoffs. <!-- compose-doc-append-sentinel: branch=fix/fix-802 pr=#816 -->
