### FEATURE: Multi-session safety — AGENT_AUTO_BRANCH enforcement (replaces heartbeat sibling detection) (2026-05-05)
Background: Multiple concurrent CC sessions on the same repo can race when committing to the default branch. The session began with a heartbeat-based detection design (per-repo .git/agent-state/<sid>.json with mtime TTL, scanned by pre-commit/pre-push). Implementation completed and tested before the user identified that heartbeats were unsuitable for multi-root VSCode workspaces — CC's cwd is the workspace's first folder, so heartbeats land in repo X while real commits go to repo Y, leaving the guard blind. The 'twisted relationship' was deemed accident-prone. Three alternatives explored: touch-based heartbeats (PostToolUse refreshes per-repo with a manifest), centralized heartbeats keyed by repo-hash, and reusing CC's transcript files as natural heartbeats. All observe a proxy rather than enforce the actual invariant. The user proposed a fourth path: eliminate the race by construction — agents never write to the default branch by default. Codex independently endorsed this strongly: comparable to GitHub's protected branches, but enforced at edit-time rather than push-time. Hard pivot. Detailed design exploration in ai-specs research-results/multi-session-design-pivot.md.
